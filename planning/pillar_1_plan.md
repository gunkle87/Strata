# Pillar 1 Plan: First Executable Fast Path

This plan normalizes the completed first-execution phase into the pillar-based
planning system.

## Pillar Goal

Deliver the first honest, deterministic, auditable real executable path for
Strata:
- admitted authored structure in `Breadboard`
- real fast-path lowering
- real executable payload export
- `Forge` load validation
- real session lifecycle
- input application
- deterministic advancement
- output reads
- public-boundary proof
- honest phase-close documentation

## Effort Model

- Pillar 1 historical score: `33`
- Target implementation-only task difficulty: `3`
- Historical task count: `11`
- Historical sizing note:
  - Pillar 1 now serves as the first real workload benchmark for future pillar
    sizing.
  - Actual implementation slices were often heavier than tiny "quick tasks".

## Task Chunks

### Task 1.1 - Lock Admitted Executable Subset
Freeze the exact admitted `FAST_4STATE` subset, primitive family, endpoint
model, connectivity rules, execution semantics, and deterministic proof shapes.

### Task 1.2 - Define Fast Executable Payload Contract
Define the first real fast-path payload contract inside the temporary shared
envelope with explicit binding and malformed-payload rules.

### Task 1.3 - Breadboard Legality Gating
Teach `Breadboard` to distinguish admitted executable structure from
placeholder-eligible or invalid requests.

### Task 1.4 - Breadboard Fast-Path Lowering
Implement deterministic lowering for the admitted executable subset.

### Task 1.5 - Descriptor Truth Binding
Bind exported descriptor identity and ordering to real lowered executable
structure instead of placeholder assumptions.

### Task 1.6 - Forge Fast Artifact Admission
Teach `Forge` to recognize, validate, and admit real fast executable artifacts
while rejecting malformed ones.

### Task 1.7 - Forge Session Lifecycle
Implement real create/reset/destroy behavior for admitted fast executable
artifacts.

### Task 1.8 - Input Application And Deterministic Advancement
Implement input application and one deterministic common advancement boundary
for the admitted fast path.

### Task 1.9 - Real Common Output Reads
Implement real output reads and minimal observation for live executable fast
sessions.

### Task 1.10 - Public-Boundary Proof
Prove the full executable path with Breadboard, Forge, and integration tests.

### Task 1.11 - Honest Phase Closure
Tighten documentation so the repo states clearly what is real, what remains
placeholder-only, and what is still deferred.
